"Chrome V8 JavaScript 엔진으로 빌드된 JavaScript 런타임입니다."

여기서 중요한 것은 'JavaScript 런타임'이라는 부분입니다.
일반적으로 런타임이라 하면 특정 언어로 개발된 프로그램을 해석하고 실행할 수 있는 환경, 즉 프로그래밍 언어가 구동되는 환경을 의미합니다.
그래서 'JavaScript 런타임'은 JS로 구현된 프로그램을 해석하고 실행할 수 있는 환경을 뜻합니다.

그 다음으로 주목해야 할 부분은 'Chrome V8 JavaScript 엔진'입니다. Node.js가 등장하기 전의 JS는 브라우저 안에서만 동작할 수 있었습니다.
왜냐하면 브라우저만이 유일하게 JS로 구현된 프로그램을 해석하고 실행할 수 있는 'JavaScript 런타임'을 내장하고 있었기 때문입니다.
2008년 구글은 MS의 인터넷 익스플로어에 대항하기 위해 크롬 브라우저를 출시합니다.
구글은 크롬 브라우저를 만들면서 V8엔진을 개발했고, 이로 인해 JS의 실행 속도가 대폭 개선됩니다.

Node.js는 구글 크롬 브라우저에 사용된 V8엔진을 이용해서 브라우저 밖에서도 JS를 실행할 수 있도록 개발된 것입니다.
그래서 Node.js는 "Chrome V8 JavaScript 엔진으로 빌드된 JavaScript 런타임"이라고 정의할 수 있었습니다.
